## The Problem

<AccordionGroup>
  <Accordion title="01 — Stuck on legacy technology">
    Most onboard loading computers still run on outdated desktop applications with interfaces designed in the 1990s. They require dedicated workstations on the bridge, proprietary software installation, and are tied to specific hardware.
  </Accordion>
  <Accordion title="02 — No real operational workflows">
    Traditional systems function as static verification tools. You enter a final condition and check whether it passes. There is no support for modelling the actual cargo operation step by step with stability verified at every intermediate stage.
  </Accordion>
  <Accordion title="03 — Ballast planning is still manual trial and error">
    The crew enters ballast quantities by hand, checks whether stability passes, adjusts, checks again. For complex operations this iterative guessing can take hours and still produce suboptimal results.
  </Accordion>
  <Accordion title="04 — No connection between shore and vessel">
    Loading plans created in the office cannot be sent directly to the onboard system. The crew re-enters everything manually.
  </Accordion>
</AccordionGroup>

## Under the Hood

<CardGroup cols={2}>
  <Card title="Latest Web Technology" icon="globe">
    A fully web-native loading computer. Runs inside a standard browser — Chrome, Firefox, Edge — without any local software installation. Served from a compact onboard hardware unit connected to the vessel's LAN. Multiple users can work simultaneously on the same worksheet.
  </Card>
  <Card title="Interactive 3D Visualisation" icon="cube">
    The vessel, its tanks, holds, and cargo are rendered in full interactive 3D directly in the browser. Fill a ballast tank and see the waterline change. Reposition a cargo item and watch the stability curve update. The 3D view is fully synchronised with table-based inputs.
  </Card>
  <Card title="Step-Based Operational Workflow" icon="list-ol">
    Model the entire operation as a sequence of steps — Load-In/Load-Out, Lift-On/Lift-Off, Float-In/Float-Out. Each step represents a discrete change and stability, longitudinal strength, and drafts are recalculated at every step.
  </Card>
  <Card title="Advanced Ballast Calculation" icon="sliders">
    Integrated ballast optimisation engine. Users define target conditions and practical constraints. The solver iterates through feasible ballast distributions to find distributions that satisfy stability, strength, and draft requirements simultaneously.
  </Card>
  <Card title="Mesh-Based Cargo Definition" icon="vector-square">
    Supports 3D mesh-based cargo definition. Instead of approximating a cargo item as a simple box, import the actual 3D mesh. The system derives volumes, centroids, and spatial envelopes directly from the geometry — essential for heavy lift, project cargo, and deck carriers.
  </Card>
  <Card title="Cargo Template Library" icon="layer-group">
    A centralised cargo template library shared across the vessel. Common cargo items are stored as reusable templates with their geometry, weight ranges, and handling properties predefined.
  </Card>
</CardGroup>

## Modules

### General Modules

| Module | Description |
|---|---|
| **Basic** | Defines loading conditions; calculates intact stability and longitudinal strength |
| **Ballast Distribution** | Determines optimal ballast tank filling levels using iterative optimisation |
| **Trim Optimization** | Recommends optimal trim for a given loading condition |
| **Online** | Real-time integration of tank level data via MODBUS communication |
| **Hoppe Ballast Control** | Interfaces directly with ballast automation systems (powered by Hoppe Marine GmbH) |
| **Damage Stability** | Performs stability and strength assessment for defined damage cases |

### Vessel-Specific Modules

| Module | Description |
|---|---|
| **General Cargo** | Loading/distribution of unitized cargo across holds and deck areas |
| **Grain / Bulk Cargo** | Loading of dry bulk and grain cargoes; evaluates grain and bulk stability requirements |
| **Container** | Container positioning, stowage planning, 3D & 2D visualization, BAPLIE data exchange |
| **DNV Container Lashing** | Integrates DNV's StowLash solver directly into the SecureLoad workflow |
| **Crane** | Simulation of cargo loading using single or tandem crane operations |
| **Jack Up** | Jack-up vessel operations with interactive 3D visualisation and leg-load distribution solver |
| **Float In / Float Off** | Float-in and float-off operations for floating docks and semi-submersible vessels |
| **Load In / Load Out** | Continuous loading and discharge operations with tide and ballast calculations |
| **Tanker** | Cargo volume and ullage reporting; ASTM and API table corrections |
| **Car Carriers** | *(Seeking First Adopter)* Precise planning of Ro-Ro cargo operations |
| **Inland Waters** | *(Seeking First Adopter)* Inland navigation vessels; ADN (2015) regulations |
| **Gas Carrier** | *(Seeking First Adopter)* Gas dispersion areas, security zones, and monitoring regions |
