# Modules

## General Modules

| Module | Description |
|---|---|
| **Basic** | Defines loading conditions; calculates intact stability and longitudinal strength |
| **Ballast Distribution** | Determines optimal ballast tank filling levels using iterative optimisation |
| **Trim Optimization** | Recommends optimal trim for a given loading condition |
| **Online** | Real-time integration of tank level data via MODBUS communication |
| **Hoppe Ballast Control** | Interfaces directly with ballast automation systems (powered by Hoppe Marine GmbH) |
| **Damage Stability** | Performs stability and strength assessment for defined damage cases |

## Vessel-Specific Modules

| Module | Description |
|---|---|
| **General Cargo** | Loading/distribution of unitized cargo across holds and deck areas |
| **Grain / Bulk Cargo** | Loading of dry bulk and grain cargoes; evaluates grain and bulk stability requirements |
| **Container** | Container positioning, stowage planning, 3D & 2D visualization, BAPLIE data exchange |
| **DNV Container Lashing** | Integrates DNV's StowLash solver directly into the SecureLoad workflow |
| **Crane** | Simulation of cargo loading using single or tandem crane operations |
| **Jack Up** | Jack-up vessel operations with interactive 3D visualisation and leg-load distribution solver |
| **Float In / Float Off** | Float-in and float-off operations for floating docks and semi-submersible vessels |
| **Load In / Load Out** | Continuous loading and discharge operations with tide and ballast calculations |
| **Tanker** | Cargo volume and ullage reporting; ASTM and API table corrections |
| **Car Carriers** | *(Seeking First Adopter)* Precise planning of Ro-Ro cargo operations |
| **Inland Waters** | *(Seeking First Adopter)* Inland navigation vessels; ADN (2015) regulations |
| **Gas Carrier** | *(Seeking First Adopter)* Gas dispersion areas, security zones, and monitoring regions |

# Deployment

| Category | Details |
|---|---|
| **Hardware & Platform** | Plug-and-play onboard hardware, Linux-based platform, no bridge workstation needed |
| **User Access** | 100% browser-based; accessible from any device on the vessel's LAN; works fully offline |
| **Data Communication** | Secured, authorized synchronization with Fleet Connect; native sensor integration via Modbus |
| **Security** | BIMCO/ICS Cyber Guidelines and IMO MSC-FAL.1/Circ.3 |
